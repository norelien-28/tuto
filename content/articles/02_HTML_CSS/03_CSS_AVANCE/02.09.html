📄 02.09 — Méthodologies CSS (BEM, SMACSS, etc.)

À mesure que les projets web grandissent, le CSS devient difficile à maintenir. Pour éviter le chaos, certaines méthodologies CSS ont été créées.
Elles apportent des règles pour structurer le code CSS de manière claire, évolutive et scalable.

🧱 Pourquoi utiliser une méthodologie CSS ?

  • Lisibilité du code

  • Éviter les conflits entre classes

  • Maintenance facilitée sur le long terme

  • Meilleure collaboration en équipe

  • Permet de réutiliser des composants efficacement

🧩 1. BEM (Block Element Modifier)

🔸 BEM = Block__Element--Modifier

  • Block : un composant autonome (ex : card)

  • Element : une partie du composant (ex : card__title)

  • Modifier : une variante du composant (ex : card--highlight)

✅ Exemple :

html
<div class="card card--highlight">
  <h2 class="card__title">Titre</h2>
  <p class="card__description">Contenu</p>
</div>

scss
.card {
  padding: 1rem;
  border: 1px solid #ccc;

  &--highlight {
    background-color: #f9f9f9;
  }

  &__title {
    font-size: 1.2rem;
  }

  &__description {
    color: #666;
  }
}

✅ Avantages :

  • Prévisible

  • Auto-documenté

  • Idéal pour du composant CSS pur

📁 2. SMACSS (Scalable and Modular Architecture for CSS)

SMACSS est une méthodologie flexible qui classe les styles en 5 catégories :

Catégorie Description
Base Styles HTML par défaut (body, h1, etc.)
Layout Grille, structure (.header, .main)
Module Composants réutilisables (.card, .btn)
State États (.is-active, .has-error)
Theme Apparence visuelle (couleurs, skins)

✅ Exemple :

html
<article class="card is-expanded theme-light">
  <h3 class="card-title">Exemple</h3>
</article>

📦 3. OOCSS (Object-Oriented CSS)

Principe : séparer structure et apparence.

Exemple :

html
<div class="box media">
  <img class="media-img" src="..." />
  <div class="media-body">Contenu</div>
</div>

  • .media : objet réutilisable

  • .media-img, .media-body : parties de l’objet

  • .box : style visuel séparé (bordure, ombre...)

✅ Idée principale :

“Traite ton HTML comme un ensemble d’objets.”

🧠 Comparatif rapide

Méthodologie Type Points forts
BEM Nomination Simple, clair, très utilisé
SMACSS Organisation Flexible, très structuré
OOCSS Approche Favorise la réutilisation à grande échelle

💡 Conseils d’utilisation

  • Choisis une seule méthodologie (ou une combinaison cohérente)

  • Documente-la pour toute l’équipe

  • Applique-la dès le début du projet

  • Combine-la avec SCSS pour tirer le meilleur des deux

🛠️ Avec SCSS + BEM

Tu peux utiliser des placeholders et mixin SCSS pour générer des classes BEM automatiquement ou structurer ton code plus facilement : 

scss
.card {
  &__title {
    font-size: 1.5rem;
  }

  &--highlight {
    background-color: yellow;
  }
}
