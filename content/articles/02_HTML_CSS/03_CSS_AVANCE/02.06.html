📄 02.06 — Introduction à Sass / SCSS

🔍 Qu’est-ce que Sass / SCSS ?

Sass (Syntactically Awesome Stylesheets) est un préprocesseur CSS qui permet d’écrire des styles plus maintenables, réutilisables et puissants.

Il ajoute des fonctionnalités au CSS comme :

  • les variables

  • les fonctions

  • les conditions

  • les boucles

  • les imports modulaire

  • les mixin, extends, etc.

Sass se décline en deux syntaxes :

Syntaxe Extension Style Exemple
Sass .sass Indentation Sans accolades, sans point-virgule
SCSS .scss Style CSS-like Proche du CSS classique

SCSS est la syntaxe la plus utilisée aujourd'hui car elle est compatible avec du CSS natif.

✅ Pourquoi utiliser Sass / SCSS ?

  • Meilleure organisation du code

  • Réutilisation facilitée (mixin, fonctions)

  • Modularité (imports, fichiers partiels)

  • Plus de lisibilité et de maintenabilité

  • Gain de temps sur des projets de grande taille

🔧 Installation

Avec npm :

bash
npm install -g sass

Vérification :

bash
sass --version

▶️ Compilation SCSS ➜ CSS

Sass doit être compilé en CSS standard. Exemple avec un fichier style.scss : 

bash
sass style.scss style.css

Compilation en "watch mode" (automatique à chaque modification) :

bash
sass --watch style.scss:style.css

🛠️ Exemple de SCSS

scss
// Déclaration de variable
$main-color: #3498db;

body {
  font-family: sans-serif;
  background-color: $main-color;

  h1 {
    font-size: 2rem;
    color: white;
  }
}

Compilation ➜

css
body {
  font-family: sans-serif;
  background-color: #3498db;
}
body h1 {
  font-size: 2rem;
  color: white;
}

🗂️ Structure classique avec SCSS

text
scss/
│
├── _variables.scss
├── _mixins.scss
├── _base.scss
├── _layout.scss
├── _components.scss
└── style.scss

Fichier principal (style.scss) :

scss
@use 'variables';
@use 'mixins';
@use 'base';
@use 'layout';
@use 'components';

🔸 Le _ dans les noms indique un fichier partiel : il ne sera pas compilé directement, mais inclus via @use ou @import.

🚧 Remarques

  • @import est déprécié dans Sass au profit de @use et @forward

  • SCSS est compatible avec tout fichier .css valide

  • Fonctionne très bien avec des outils comme Webpack, Vite, Parcel, etc.
