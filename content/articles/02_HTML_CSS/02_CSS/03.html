2.2.3 – Responsive Design (Design Adaptatif)

Introduction

Le responsive design permet à un site web de s’adapter automatiquement à la taille de l’écran : ordinateur, tablette, smartphone...
C’est essentiel aujourd’hui pour garantir une bonne expérience utilisateur sur tous les appareils.

1. Pourquoi faire du responsive ?

  • Les utilisateurs consultent les sites sur des écrans très variés.

  • Un site non adapté peut être difficile à lire ou à naviguer.

  • Le responsive améliore le référencement SEO.

  • C’est une pratique standard du web moderne.

2. Les bases du responsive : la meta viewport

Dans le <head> de ta page HTML, ajoute cette balise pour que le navigateur gère correctement le zoom et la largeur : 

html
<meta name="viewport" content="width=device-width, initial-scale=1.0">

3. Les media queries

Ce sont des règles CSS qui s’appliquent seulement sous certaines conditions, notamment la largeur de l’écran.

Syntaxe :

css
@media (max-width: 768px) {
  /* Styles pour écran ≤ 768px */
}

Exemple :

css
body {
  background-color: white;
  font-size: 18px;
}

@media (max-width: 768px) {
  body {
    background-color: lightgray;
    font-size: 16px;
  }
}

4. Utilisation pratique

4.1 Adapter la mise en page

Par exemple, passer d’une disposition en colonnes sur desktop à une colonne unique sur mobile. 

css
.main-container {
  display: flex;
  flex-direction: row;
}

@media (max-width: 768px) {
  .main-container {
    flex-direction: column;
  }
}

4.2 Modifier la taille des textes, boutons, images

css
h1 {
  font-size: 36px;
}

@media (max-width: 480px) {
  h1 {
    font-size: 24px;
  }
}

5. Exemple complet

HTML :

html
<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Responsive Demo</title>
  <link rel="stylesheet" href="style.css" />
</head>
<body>
  <header>
    <h1>Mon site responsive</h1>
  </header>
  <main class="main-container">
    <aside class="sidebar">Menu</aside>
    <section class="content">
      <p>Contenu principal ici.</p>
    </section>
  </main>
  <footer>© 2025</footer>
</body>
</html>

CSS (style.css) :

css
body {
  font-family: Arial, sans-serif;
  margin: 0; padding: 0;
}

.main-container {
  display: flex;
  flex-direction: row;
  gap: 20px;
  padding: 20px;
}

.sidebar {
  background-color: #ddd;
  width: 200px;
  padding: 15px;
}

.content {
  flex: 1;
  background-color: #eee;
  padding: 15px;
}

/* Responsive */
@media (max-width: 768px) {
  .main-container {
    flex-direction: column;
  }

  .sidebar {
    width: 100%;
  }
}

6. Bonnes pratiques

  • Toujours inclure la meta viewport.

  • Tester sur plusieurs tailles d’écran.

  • Utiliser des unités relatives (em, %, vh, vw) plutôt que fixes (px).

  • Éviter de cacher trop de contenu sur mobile.

  • Garder une navigation simple et accessible.

7. Exercices

  1. Ajoute une media query pour changer la couleur de fond sur écran < 480px.

  2. Sur mobile, cache la barre latérale (display: none;).

  3. Modifie la taille des polices pour améliorer la lisibilité.

  4. Teste la page sur différents appareils ou outils de simulation (devtools).

8. Conclusion

Le responsive design est indispensable pour créer des sites modernes et accessibles partout.
Avec les media queries et des mises en page flexibles, tu peux maîtriser l’adaptation à tous les écrans.
