2.4.5 – Gestion des Événements et des Formulaires dans React

🖱️ 1. Gestion des événements

React gère les événements de manière similaire à JavaScript, mais avec quelques différences :

✅ Principales différences :

  • Les noms d’événements sont en camelCase (onClick, onChange, etc.)

  • On passe des fonctions en tant que gestionnaires, pas des chaînes de caractères.

📌 Exemple simple :

jsx
function Bouton() {
  function handleClick() {
    alert('Bouton cliqué !');
  }

  return <button onClick={handleClick}>Cliquez-moi</button>;
}

Pas de parenthèses après handleClick, sinon il s’exécute immédiatement !

📤 2. Passage de paramètres à un événement

jsx
function Bouton({ message }) {
  function handleClick() {
    alert(message);
  }

  return <button onClick={handleClick}>Alerte</button>;
}

Ou bien avec des fonctions anonymes :

jsx
<button onClick={() => handleClick('Coucou !')}>Clic</button>

📝 3. Formulaires contrôlés

Dans React, on utilise généralement des formulaires contrôlés, où l’état du champ est synchronisé avec le state React.

📋 Exemple avec un champ texte :

jsx
import { useState } from 'react';

function Formulaire() {
  const [nom, setNom] = useState('');

  function handleChange(event) {
    setNom(event.target.value);
  }

  function handleSubmit(event) {
    event.preventDefault();
    alert(`Nom soumis : ${nom}`);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input type="text" value={nom} onChange={handleChange} />
      <button type="submit">Envoyer</button>
    </form>
  );
}

🔁 4. Champs multiples

Pour gérer plusieurs champs, on peut utiliser un objet state :

jsx
const [formData, setFormData] = useState({ nom: '', email: '' });

function handleChange(event) {
  const { name, value } = event.target;
  setFormData(prevData => ({
    ...prevData,
    [name]: value,
  }));
}

Et dans les inputs :

jsx
<input name="nom" value={formData.nom} onChange={handleChange} />
<input name="email" value={formData.email} onChange={handleChange} />

✅ 5. Bonnes pratiques

  • Toujours lier les champs de formulaire à un state.

  • Utiliser event.preventDefault() pour éviter le rechargement de page.

  • Nommer les champs avec name="" pour pouvoir les identifier.

🧠 6. Résumé des événements courants

Événement React Usage courant
onClick Clic sur un bouton ou lien
onChange Modification d’un champ de formulaire
onSubmit Soumission d’un formulaire
onMouseEnter Passage de la souris
onKeyDown Appui sur une touche
