2.4.1 – Introduction à React

🚀 Qu’est-ce que React ?

React est une bibliothèque JavaScript développée par Facebook (Meta), utilisée pour construire des interfaces utilisateur (UI). Elle permet de créer des composants réutilisables qui reflètent l’état de l’application de manière dynamique.

React est souvent utilisé pour développer des applications web SPA (Single Page Applications) performantes et interactives.

🧠 Pourquoi utiliser React ?

✅ Les avantages :

  • Composants réutilisables : chaque partie de l’interface est un composant indépendant.

  • Virtual DOM : React met à jour uniquement les parties du DOM qui changent, ce qui le rend très rapide.

  • Unidirectional Data Flow : la gestion de données se fait de parent à enfant, ce qui simplifie le suivi de l’état.

  • Écosystème riche : avec des outils comme Redux, React Router, Next.js, etc.

  • Large communauté : beaucoup de ressources, d’outils, et de support.

🧩 React vs Framework

Bien que souvent comparé à des frameworks comme Angular ou Vue.js, React est une bibliothèque, centrée uniquement sur la vue (View) dans le modèle MVC.

Il ne s’occupe ni du routage, ni de la gestion d’état globale par défaut, ni du backend.

🏗️ Concepts fondamentaux de React

Concept Description
Composants Blocs de code réutilisables qui retournent du JSX (HTML-like).
JSX Une extension de JavaScript qui permet d’écrire du HTML dans le JS.
Props Les propriétés passées aux composants.
State L’état interne d’un composant, qui déclenche un re-render en cas de changement.
Hooks Fonctions permettant d’utiliser le state et d'autres fonctionnalités dans les composants fonctionnels.

📦 React en un coup d'œil

jsx
import React from 'react';

function Bonjour(props) {
  return <h1>Bonjour, {props.nom} !</h1>;
}

export default function App() {
  return <Bonjour nom="Jean" />;
}

🔍 Explication :

  • Bonjour est un composant.

  • Il utilise une prop nom pour afficher une valeur dynamique.

  • App est le composant principal qui compose Bonjour.
