2.4.3 – Composants et Props dans React

🧱 1. Qu’est-ce qu’un composant ?

Un composant React est une fonction ou une classe JavaScript qui retourne du JSX (du HTML dans du JS). C’est l’unité de base d’une interface React.

Il existe deux types principaux :

  • Fonctionnels (modernes, avec hooks)

  • Classiques (plus anciens, rarement utilisés aujourd'hui)

🔧 Exemple d’un composant fonctionnel :

jsx
function Bonjour() {
  return <h1>Bonjour, monde !</h1>;
}

🧩 Utilisation dans un autre composant :

jsx
function App() {
  return (
    <div>
      <Bonjour />
    </div>
  );
}

📦 2. JSX : JavaScript + HTML

Le JSX permet d’écrire du HTML dans du JavaScript.

Exemple :

jsx
const element = <h1>Hello, React!</h1>;

Derrière le JSX, React utilise React.createElement() pour générer des éléments.

📨 3. Les Props (Properties)

Les props permettent de transmettre des données d’un composant parent à un composant enfant.

Exemple :

jsx
function Bonjour(props) {
  return <h1>Bonjour, {props.nom} !</h1>;
}

function App() {
  return <Bonjour nom="Jean" />;
}

nom="Jean" est une prop passée à Bonjour. Dans le composant, on y accède via props.nom.

📌 4. Props : quelques règles

  • Les props sont en lecture seule (read-only).

  • Elles permettent de rendre les composants dynamiques et réutilisables.

  • On peut transmettre n’importe quel type de données (string, number, objet, fonction, etc.).

Exemple avec différents types :

jsx
<MonComposant age={30} actif={true} données={{ nom: "Alice" }} />

♻️ 5. Réutilisabilité avec les props

jsx
function Titre(props) {
  return <h1>{props.texte}</h1>;
}

function App() {
  return (
    <>
      <Titre texte="Accueil" />
      <Titre texte="À propos" />
      <Titre texte="Contact" />
    </>
  );
}

Un seul composant Titre, utilisé 3 fois avec des contenus différents.

🧠 6. Destructuration des props

Pour rendre le code plus clair :

jsx
function Bonjour({ nom }) {
  return <h1>Salut {nom} !</h1>;
}

🚫 7. Props vs State

Aspect Props State
Définition Données passées au composant Données internes au composant
Modifiable ? ❌ Non (read-only) ✅ Oui (modifiable via useState)
Utilisation Communication parent → enfant Gestion dynamique de l’état
