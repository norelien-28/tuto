08.07 - Ajouter des styles et scripts correctement

Introduction

Dans WordPress, il est déconseillé d'ajouter directement vos fichiers CSS ou JavaScript dans le fichier header.php via des balises <link> ou <script>.
La méthode recommandée est d'utiliser les fonctions wp_enqueue_style() et wp_enqueue_script(), dans le fichier functions.php.

Cela permet :

  • Une meilleure gestion des dépendances

  • Une compatibilité avec les plugins et thèmes enfants

  • Le respect des bonnes pratiques WordPress

1. Exemple de structure de thème

Ton thème devrait contenir :

pgsql
mon-theme/
├── css/
│   └── style.css
├── js/
│   └── script.js
├── functions.php
├── index.php
├── header.php
├── footer.php
└── style.css

2. Ajouter un fichier CSS avec wp_enqueue_style()

Dans ton fichier functions.php :

php
<?php
function mon_theme_enqueue_styles() {
  wp_enqueue_style(
    'mon-style-principal', // identifiant du style
    get_template_directory_uri() . '/css/style.css', // chemin vers le fichier
    array(), // dépendances (ex: Bootstrap)
    '1.0', // version
    'all' // media
  );
}
add_action('wp_enqueue_scripts', 'mon_theme_enqueue_styles');

Tu peux également charger le fichier style.css situé à la racine du thème : 

php
wp_enqueue_style('style', get_stylesheet_uri());

3. Ajouter un fichier JavaScript avec wp_enqueue_script()

Toujours dans functions.php :

php
function mon_theme_enqueue_scripts() {
  wp_enqueue_script(
    'mon-script-principal',
    get_template_directory_uri() . '/js/script.js',
    array('jquery'), // dépendances
    '1.0',
    true // charger dans le footer
  );
}
add_action('wp_enqueue_scripts', 'mon_theme_enqueue_scripts');

💡 L’argument true permet de charger le script juste avant la fermeture de </body>, ce qui est plus performant.

4. Charger des bibliothèques externes (CDN)

Exemple : ajouter Bootstrap CSS depuis un CDN :

php
wp_enqueue_style(
  'bootstrap',
  'https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css'
);

Et Bootstrap JS :

php
wp_enqueue_script(
  'bootstrap',
  'https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js',
  array('jquery'),
  null,
  true
);

5. Regrouper les appels

Tu peux centraliser les deux fonctions comme ceci :

php
function mon_theme_assets() {
  // CSS
  wp_enqueue_style('style', get_stylesheet_uri());
  wp_enqueue_style('bootstrap', 'https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css');

  // JS
  wp_enqueue_script('bootstrap', 'https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js', array('jquery'), null, true);
  wp_enqueue_script('main', get_template_directory_uri() . '/js/script.js', array(), '1.0', true);
}
add_action('wp_enqueue_scripts', 'mon_theme_assets');

6. Ne pas faire : les mauvaises pratiques

❌ À éviter :

  • Ajouter manuellement des <script> ou <link> dans header.php

  • Oublier de préciser les dépendances (comme jQuery)

  • Ne pas charger les fichiers au bon endroit (ex : JS dans le <head>)

Conclusion

Charger les scripts et styles correctement avec wp_enqueue_style() et wp_enqueue_script() garantit la compatibilité de ton thème avec l’écosystème WordPress. Cela évite aussi les conflits avec d’autres thèmes ou plugins.
