08.15 – Sécuriser son site WordPress

Introduction

WordPress est le CMS le plus utilisé au monde… donc aussi l’un des plus ciblés par les attaques.
Heureusement, en appliquant quelques bonnes pratiques, tu peux fortement réduire les risques de piratage.

1. Tenir WordPress à jour

La mise à jour régulière de :

  • WordPress (cœur)

  • Thèmes

  • Plugins

… est le premier réflexe à adopter. Les mises à jour corrigent souvent des failles de sécurité.

✅ Active les mises à jour automatiques si possible.

2. Utiliser des plugins fiables

  • Choisis des plugins populaires et bien notés.

  • Évite les plugins abandonnés ou peu téléchargés.

  • Supprime ceux que tu n’utilises plus.

⚠️ N’utilise jamais de plugins ou thèmes piratés (nulled) : ils sont souvent infectés.

3. Changer l’URL de connexion

Par défaut, l’accès à l’administration est disponible via /wp-admin.
Tu peux le modifier avec un plugin comme :

  • WPS Hide Login

  • iThemes Security

Cela évite les attaques automatisées sur la page de login.

4. Utiliser des mots de passe forts

  • Utilise un mot de passe unique et complexe pour chaque utilisateur.

  • Active la double authentification (2FA) avec des plugins comme Wordfence ou WP 2FA.

5. Gérer les utilisateurs avec précaution

  • Ne donne le rôle administrateur qu’aux personnes qui en ont réellement besoin.

  • Supprime les comptes inactifs ou suspects.

6. Installer un plugin de sécurité

Quelques plugins de sécurité fiables :

  • Wordfence Security – Firewall, scans, alertes

  • iThemes Security – Facile à configurer

  • Sucuri Security – Audit, monitoring

Ces plugins peuvent aussi bloquer les tentatives de force brute.

7. Sauvegardes régulières

Fais des sauvegardes automatiques :

  • Base de données

  • Fichiers WordPress

Plugins recommandés :

  • UpdraftPlus

  • WPvivid

  • BlogVault (freemium)

8. Désactiver l’édition de fichiers

Ajoute cette ligne dans le fichier wp-config.php pour empêcher l’édition directe depuis l’admin : 

php
define('DISALLOW_FILE_EDIT', true);

9. Restreindre l’accès au tableau de bord

Tu peux limiter l'accès au tableau de bord (/wp-admin) aux administrateurs avec du code ou un plugin.

10. Vérifier les fichiers sensibles

  • wp-config.php : doit être en lecture seule

  • .htaccess : vérifie qu’il n’a pas été modifié

  • Supprime les fichiers inutiles comme readme.html

Conclusion

Sécuriser un site WordPress n’est pas compliqué, mais demande un minimum de rigueur.
Avec quelques précautions, tu peux protéger ton site contre la majorité des attaques courantes.
