08.17 – Déployer un site WordPress en ligne

Introduction

Déployer un site WordPress en ligne consiste à transférer ton site depuis un environnement local (ordinateur personnel) vers un hébergement web accessible publiquement. Cette étape est cruciale pour rendre ton site visible par tous sur Internet.

1. Préparer son site local pour le déploiement

  • Vérifie que tout fonctionne correctement (thèmes, plugins, contenus).

  • Effectue une sauvegarde complète (fichiers + base de données).

  • Nettoie le site en supprimant les éléments inutiles (plugins, thèmes, médias).

2. Choisir un hébergement web

  • Hébergement mutualisé : économique, adapté aux petits sites.

  • Hébergement VPS/dédié : plus de contrôle et performance.

  • Hébergement spécialisé WordPress : optimisé pour WP, avec services inclus.

3. Transférer les fichiers

3.1 Via FTP/SFTP

  • Utilise un client FTP (FileZilla, Cyberduck, WinSCP).

  • Connecte-toi à ton hébergement.

  • Transfère tous les fichiers WordPress (depuis le dossier local) vers le dossier racine du serveur (souvent public_html).

3.2 Via cPanel ou gestionnaire de fichiers

  • Certains hébergeurs proposent un gestionnaire de fichiers dans leur panneau de contrôle.

  • Tu peux y uploader un archive ZIP de ton site et l’extraire directement.

4. Exporter et importer la base de données

4.1 Exporter la base locale

  • Utilise phpMyAdmin ou un outil équivalent.

  • Export au format SQL de la base WordPress.

4.2 Importer sur le serveur

  • Crée une base de données sur ton hébergement via cPanel ou interface.

  • Importe le fichier SQL via phpMyAdmin ou outil similaire.

5. Modifier le fichier wp-config.php

  • Mets à jour les informations de connexion à la base de données (DB_NAME, DB_USER, DB_PASSWORD, DB_HOST).

  • Vérifie les autres constantes si nécessaire.

6. Mettre à jour les URLs du site

  • Si l’URL locale et celle en ligne diffèrent, il faut mettre à jour la base pour refléter la nouvelle adresse.

  • Plusieurs méthodes :

    • Utiliser un plugin comme Better Search Replace pour remplacer l’URL locale par l’URL du site en ligne.

    • Exécuter une requête SQL via phpMyAdmin.

    • Modifier directement les options siteurl et home dans la table wp_options.

7. Vérifications post-déploiement

  • Teste le site en ligne sur différentes pages.

  • Vérifie les liens, les médias, les formulaires.

  • Teste les fonctionnalités de connexion et d’administration.

  • Supprime ou protège les fichiers sensibles (ex : wp-config.php).

8. Configurations supplémentaires possibles

  • Installer un certificat SSL (HTTPS).

  • Configurer les sauvegardes automatiques.

  • Mettre en place des outils de monitoring et sécurité.

  • Optimiser la performance (cache, CDN).

Conclusion

Le déploiement d’un site WordPress en ligne demande une préparation rigoureuse, une bonne maîtrise des outils de transfert et une attention particulière aux configurations. Une fois réalisé correctement, ton site sera accessible, sécurisé et prêt à accueillir ses visiteurs.
