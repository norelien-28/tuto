08.25 – Créer des endpoints personnalisés

Introduction

L’API REST de WordPress peut être étendue en créant des endpoints personnalisés. Cela permet d’exposer des données spécifiques ou des fonctionnalités non couvertes par l’API native, adaptées aux besoins de votre projet.

1. Pourquoi créer un endpoint personnalisé ?

  • Pour fournir des données ou services spécifiques à votre site.

  • Pour intégrer des fonctionnalités personnalisées dans des applications externes.

  • Pour sécuriser ou restreindre l’accès à certaines données.

2. Comment créer un endpoint personnalisé ?

2.1 Utiliser le hook rest_api_init

Ajoutez votre code dans le fichier functions.php de votre thème enfant ou dans un plugin personnalisé : 

php
add_action('rest_api_init', function () {
    register_rest_route('monplugin/v1', '/monendpoint', array(
        'methods' => 'GET',
        'callback' => 'ma_fonction_callback',
        'permission_callback' => '__return_true', // à adapter pour la sécurité
    ));
});

function ma_fonction_callback(WP_REST_Request $request) {
    return new WP_REST_Response(array(
        'message' => 'Hello depuis mon endpoint personnalisé !',
    ), 200);
}

3. Paramètres et méthodes HTTP

  • Les endpoints peuvent accepter des paramètres via l’objet $request.

  • Support des méthodes GET, POST, PUT, DELETE selon l’action souhaitée.

Exemple de récupération d’un paramètre :

php
function ma_fonction_callback(WP_REST_Request $request) {
    $param = $request->get_param('id');
    // Traitement avec $param
}

4. Sécurité

  • Toujours définir permission_callback pour contrôler l’accès.

  • Exemple basique pour autoriser uniquement les utilisateurs connectés :

php
'permission_callback' => function () {
    return is_user_logged_in();
}

  • Pour des besoins avancés, gérer les rôles et capacités.

5. Tester son endpoint

  • Utiliser des outils comme Postman, Insomnia ou curl.

  • Exemple curl :

bash
curl https://votresite.com/wp-json/monplugin/v1/monendpoint

Conclusion

Créer des endpoints personnalisés dans l’API REST de WordPress permet d’adapter votre site aux besoins spécifiques d’applications externes tout en gardant la sécurité et la flexibilité.
