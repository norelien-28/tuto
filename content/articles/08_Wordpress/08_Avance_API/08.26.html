08.26 – Headless WordPress : utiliser WP avec React/Vue

Introduction

Le mode headless consiste à utiliser WordPress uniquement comme système de gestion de contenu (CMS), tandis que le frontend est développé avec un framework moderne comme React ou Vue.js. La communication se fait via l’API REST.

1. Pourquoi utiliser WordPress en mode headless ?

  • Séparation claire entre contenu et présentation.

  • Utiliser des technologies frontend modernes pour une meilleure expérience utilisateur.

  • Faciliter l’intégration multi-plateformes (web, mobile, IoT).

  • Optimiser les performances grâce à des frameworks SPA (Single Page Application).

2. Architecture générale

  • WordPress gère uniquement le contenu et les données via son API REST.

  • React ou Vue consomme les données JSON et affiche l’interface utilisateur.

  • Le frontend peut être hébergé séparément, même en statique via un générateur de site (ex : Next.js, Nuxt.js).

3. Mise en place de base

3.1 Préparer WordPress

  • Installer et configurer l’API REST (native depuis WP 4.7).

  • Ajouter des endpoints personnalisés si besoin.

  • Gérer l’authentification pour les actions sécurisées.

3.2 Créer le frontend

  • Initier un projet React ou Vue.

  • Utiliser fetch ou une bibliothèque comme Axios pour interroger l’API WP.

  • Afficher les contenus (articles, pages, CPT) récupérés en JSON.

4. Exemple simple avec React

jsx
import React, { useEffect, useState } from 'react';

function App() {
  const [posts, setPosts] = useState([]);

  useEffect(() => {
    fetch('https://votresite.com/wp-json/wp/v2/posts')
      .then(response => response.json())
      .then(data => setPosts(data));
  }, []);

  return (
    <div>
      <h1>Articles WordPress</h1>
      <ul>
        {posts.map(post => (
          <li key={post.id} dangerouslySetInnerHTML={{ __html: post.title.rendered }} />
        ))}
      </ul>
    </div>
  );
}

export default App;

5. Bonnes pratiques

  • Gérer le SEO (prérendu, SSR, ou solutions spécifiques).

  • Sécuriser l’API et les données sensibles.

  • Optimiser la gestion des médias.

  • Prévoir la pagination, filtrage et recherche côté frontend.

Conclusion

Utiliser WordPress en mode headless avec React ou Vue est une approche moderne qui combine la puissance d’un CMS mature avec la flexibilité des frameworks frontend actuels, offrant ainsi une expérience utilisateur riche et performante.
