08.10 – Créer des templates pour les CPT (Custom Post Types)

Introduction

Quand tu crées un Custom Post Type (CPT) dans WordPress, tu peux aller plus loin en créant des templates personnalisés pour :

  • afficher un projet individuel (single-projet.php)

  • afficher la liste de tous les projets (archive-projet.php)

Cela te permet de contrôler entièrement l’apparence de ton contenu personnalisé dans le thème.

1. Rappel : nom du CPT

Imaginons que tu as créé un CPT nommé projet.

Tu dois donc créer deux fichiers dans ton thème :

  • single-projet.php → pour un projet individuel

  • archive-projet.php → pour la page d’archive (liste)

Ces noms doivent correspondre au slug du CPT (défini dans register_post_type()).

2. Créer le fichier single-projet.php

Ce fichier gère l’affichage d’un projet individuel.

Exemple de base :

php
<?php get_header(); ?>

<main>
  <?php if ( have_posts() ) : while ( have_posts() ) : the_post(); ?>

    <h1><?php the_title(); ?></h1>
    <div>
      <?php the_content(); ?>
    </div>

    <?php
    $client = get_field('client');
    if ($client) {
      echo '<p><strong>Client :</strong> ' . esc_html($client) . '</p>';
    }
    ?>

  <?php endwhile; endif; ?>
</main>

<?php get_footer(); ?>

3. Créer le fichier archive-projet.php

Ce fichier gère la liste de tous les projets (archive).

Il s’affichera automatiquement à l’adresse :

arduino
https://monsite.com/projets

(si tu as activé 'has_archive' => true et défini 'rewrite' => ['slug' => 'projets'] dans register_post_type()).

Exemple de base :

php
<?php get_header(); ?>

<main>
  <h1>Nos Projets</h1>

  <?php if ( have_posts() ) : while ( have_posts() ) : the_post(); ?>
    <article>
      <h2><a href="<?php the_permalink(); ?>"><?php the_title(); ?></a></h2>
      <p><?php the_excerpt(); ?></p>
    </article>
  <?php endwhile; else : ?>
    <p>Aucun projet trouvé.</p>
  <?php endif; ?>
</main>

<?php get_footer(); ?>

4. Forcer l’actualisation des permaliens

⚠️ Après avoir ajouté ces fichiers, rends-toi dans
Réglages > Permaliens et clique sur "Enregistrer les modifications",
même sans rien changer. Cela permet de reconstruire les règles d’URL.

5. Gérer l’ordre ou le tri des projets

Tu peux personnaliser la requête avec des filtres ou via une requête manuelle (WP_Query) si besoin.
Exemple : trier les projets par un champ ACF ou par date personnalisée.

6. Créer un template spécifique à un seul projet

Tu peux aussi créer un template par slug ou ID :

  • single-projet-slug.php (ex : single-projet-site-ecommerce.php)

  • single-projet-42.php (pour l’ID 42)

Cela permet de faire une mise en page unique pour certains projets.

Conclusion

Créer des templates pour les Custom Post Types te permet de totalement contrôler la présentation de tes types de contenu personnalisés.
Tu peux ainsi créer des interfaces propres, cohérentes et adaptées aux besoins de ton client ou projet.
