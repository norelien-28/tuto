08.09 – Utiliser Advanced Custom Fields (ACF)

Introduction

ACF (Advanced Custom Fields) est un plugin WordPress qui permet d’ajouter facilement des champs personnalisés à tes contenus (articles, pages, CPT…).
Il est idéal pour transformer WordPress en un CMS puissant et flexible, sans avoir à coder une interface d’administration complexe.

1. Installer le plugin ACF

Tu peux l’installer de deux façons :

Depuis l’administration WordPress :

  • Va dans Extensions > Ajouter

  • Recherche Advanced Custom Fields

  • Clique sur Installer puis Activer

Via Composer (optionnel) :

Si tu utilises Composer dans ton thème ou projet :

bash
composer require advanced-custom-fields/advanced-custom-fields

2. Créer un groupe de champs

Une fois ACF installé :

  1. Va dans Custom Fields > Add New

  2. Donne un nom à ton groupe (ex. : Informations projet)

  3. Clique sur + Add Field

    • Choisis un type (texte, image, lien, WYSIWYG…)

    • Nomme le champ (ex : lien_projet, client, date_debut)

  4. Dans Location, choisis où ce groupe de champs doit apparaître :

    • Exemple : Post Type is égal à "Projet"

  5. Publie le groupe de champs

3. Afficher les champs dans ton thème

Exemple : afficher un champ texte client

Dans le fichier single-projet.php ou template-parts/projet.php : 

php
<?php
$client = get_field('client');

if ($client) {
  echo '<p><strong>Client :</strong> ' . esc_html($client) . '</p>';
}
?>

Autres exemples de types de champs

  • Image :

php
$image = get_field('image_du_projet');
if ($image) {
  echo '<img src="' . esc_url($image['url']) . '" alt="' . esc_attr($image['alt']) . '">';
}

  • Lien :

php
$lien = get_field('lien_projet');
if ($lien) {
  echo '<a href="' . esc_url($lien['url']) . '" target="_blank">' . esc_html($lien['title']) . '</a>';
}

4. Options avancées utiles

  • Repeater (champ répété : liste d’éléments) → disponible en version Pro

  • Flexible Content : mise en page dynamique

  • Champs conditionnels : affichés selon certaines valeurs

  • Local JSON : sauvegarde automatique des champs dans ton thème (idéal pour Git)

5. Pourquoi utiliser ACF ?

✅ Simple à configurer
✅ Aucun code pour l’interface admin
✅ Compatible avec les CPT
✅ Puissant pour des contenus structurés
✅ Idéal pour les développeurs ET les clients

6. Alternatives à ACF

Bien qu’ACF soit le plus populaire, tu peux aussi explorer :

  • Meta Box

  • Pods

  • CMB2

  • Ou l’API native add_meta_box() (plus complexe)

Conclusion

ACF est l’un des outils les plus puissants pour personnaliser WordPress sans sacrifier la maintenabilité. Il te permet de séparer clairement le contenu de la présentation, et de rendre le back-office plus intuitif pour les utilisateurs finaux.
