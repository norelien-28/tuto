08.12 – Hooks : actions et filtres

Introduction

Les hooks (crochets) sont l’un des mécanismes les plus puissants de WordPress.
Ils te permettent de modifier ou d’étendre le comportement de WordPress sans modifier son cœur.

Il existe deux types principaux de hooks :

  • Actions : pour exécuter du code à un moment donné

  • Filtres : pour modifier des données avant qu’elles ne soient utilisées ou affichées

1. Les actions (add_action)

Les actions te permettent d’exécuter une fonction à un moment précis du cycle de WordPress.

Syntaxe :

php
add_action('nom_hook', 'nom_de_ta_fonction');

Exemple simple :

php
function afficher_message_admin() {
    echo '<div class="notice notice-success"><p>Plugin activé avec succès !</p></div>';
}
add_action('admin_notices', 'afficher_message_admin');

Ce code affiche un message dans l’administration.

2. Les filtres (add_filter)

Les filtres permettent de modifier une donnée avant qu’elle ne soit utilisée.

Syntaxe :

php
add_filter('nom_hook', 'nom_de_ta_fonction');

Exemple simple :

php
function prefixer_titre($titre) {
    return '★ ' . $titre;
}
add_filter('the_title', 'prefixer_titre');

Ce filtre ajoute une étoile devant chaque titre d’article.

3. Priorité et nombre d’arguments

Tu peux spécifier une priorité (plus elle est basse, plus ta fonction est appelée tôt), et le nombre d’arguments que ta fonction peut recevoir. 

php
add_filter('the_title', 'prefixer_titre', 20, 1);

4. Créer tes propres hooks

Tu peux aussi définir des hooks personnalisés dans ton propre code, que d'autres développeurs pourront utiliser :

Action personnalisée :

php
do_action('mon_hook_perso');

Filtre personnalisé :

php
$valeur = apply_filters('mon_filtre_perso', $valeur);

5. Où trouver la liste des hooks ?

Tu peux consulter :

  • https://developer.wordpress.org/reference/

  • Le code source de WordPress (fichier wp-includes/)

  • Les documentations de thèmes et plugins que tu utilises

6. Bonnes pratiques

  • Toujours nommer tes fonctions avec un préfixe pour éviter les conflits (ex : monplugin_)

  • Évite de surcharger des hooks sans savoir ce qu’ils font

  • Teste bien les filtres : ils peuvent affecter tout le site si mal utilisés

Conclusion

Comprendre et utiliser les actions et filtres est indispensable pour tout développeur WordPress.
Ils sont la clé de l’extensibilité, que ce soit pour les thèmes, plugins ou personnalisation du cœur de WP.
