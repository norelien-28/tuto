08.14 – Personnaliser les rôles et permissions

Introduction

WordPress repose sur un système de rôles et capacités (capabilities) pour gérer les droits d’accès des utilisateurs.
Personnaliser ces rôles permet de contrôler précisément ce que chaque utilisateur peut ou ne peut pas faire dans l’administration de ton site.

1. Les rôles par défaut dans WordPress

WordPress propose plusieurs rôles intégrés :

Rôle Description principale
Abonné Lire du contenu, modifier son profil
Contributeur Écrire des articles mais ne peut pas les publier
Auteur Publier et gérer ses propres articles
Éditeur Gérer tous les articles et pages
Administrateur Contrôle total sur le site

2. Ajouter un rôle personnalisé

Tu peux créer un rôle avec add_role() :

php
add_role('client', 'Client', [
  'read' => true,
  'edit_posts' => false,
  'delete_posts' => false,
]);

Ce code ajoute un rôle « Client » qui peut uniquement lire du contenu.

🔁 Pour éviter les doublons, assure-toi que ce code s’exécute une seule fois (ex. : lors de l’activation d’un plugin).

3. Ajouter ou retirer des capacités

Tu peux ajouter ou retirer des permissions à un rôle existant avec l’objet WP_Role.

Ajouter une capacité :

php
$role = get_role('editor');
$role->add_cap('edit_theme_options');

Retirer une capacité :

php
$role = get_role('author');
$role->remove_cap('publish_posts');

4. Donner une capacité à un utilisateur spécifique

Tu peux aussi modifier les permissions d’un utilisateur en particulier :

php
$user = get_user_by('login', 'jean');
$user->add_cap('edit_pages');

5. Vérifier une capacité (dans ton code)

Pour restreindre l’accès à une partie de l’interface ou d’un shortcode, utilise :

php
if ( current_user_can('edit_pages') ) {
  echo 'Tu peux modifier les pages !';
}

6. Plugins utiles pour gérer les rôles

Si tu ne veux pas manipuler le code :

  • 🔌 Members (par MemberPress) – Très complet

  • 🔌 User Role Editor – Facile à utiliser et populaire

7. Supprimer un rôle personnalisé

Si tu veux retirer un rôle ajouté :

php
remove_role('client');

8. Liste de capacités utiles

Voici quelques capacités que tu peux attribuer :

  • edit_posts

  • edit_pages

  • edit_others_posts

  • publish_posts

  • delete_posts

  • manage_options (accès aux réglages)

  • edit_theme_options (accès à l’apparence)

  • upload_files (envoyer des fichiers)

Tu peux retrouver la liste complète dans la doc officielle WordPress.

Conclusion

En personnalisant les rôles et permissions, tu peux rendre ton site WordPress plus sécurisé, adapté à ton équipe et plus professionnel.
C’est particulièrement utile pour les sites clients ou les projets collaboratifs.
