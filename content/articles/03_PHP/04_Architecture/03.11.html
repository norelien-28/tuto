3.2.11 – Sécurité : Injection SQL, XSS, CSRF (bases)

1. Injection SQL

Qu’est-ce que c’est ?

L’injection SQL permet à un attaquant d’insérer du code SQL malveillant dans une requête, compromettant la base de données.

Exemple vulnérable

php
$id = $_GET['id'];
$query = "SELECT * FROM utilisateurs WHERE id = $id";
$result = $pdo->query($query);

Si id vaut 1; DROP TABLE utilisateurs, la table sera supprimée.

Protection

  • Utiliser des requêtes préparées avec paramètres liés.

php
$stmt = $pdo->prepare('SELECT * FROM utilisateurs WHERE id = :id');
$stmt->execute(['id' => $id]);

2. XSS (Cross-Site Scripting)

Qu’est-ce que c’est ?

Injection de scripts malveillants dans les pages vues par les utilisateurs.

Exemple vulnérable

Afficher directement une donnée utilisateur sans nettoyage :

php
echo $_GET['nom'];

Si nom contient du code JavaScript, il s’exécutera.

Protection

  • Échapper les données avec htmlspecialchars() :

php
echo htmlspecialchars($_GET['nom']);

3. CSRF (Cross-Site Request Forgery)

Qu’est-ce que c’est ?

Un attaquant force un utilisateur authentifié à exécuter une action non désirée.

Protection

  • Utiliser des jetons CSRF (tokens) dans les formulaires.

Exemple simplifié

Dans le formulaire :

php
session_start();
$token = bin2hex(random_bytes(32));
$_SESSION['csrf_token'] = $token;
?>

<form method="post" action="traitement.php">
    <input type="hidden" name="csrf_token" value="<?= $token ?>">
    <!-- autres champs -->
</form>

Dans traitement.php :

php
session_start();

if (!isset($_POST['csrf_token']) || $_POST['csrf_token'] !== $_SESSION['csrf_token']) {
    die('Erreur CSRF');
}

// traitement sécurisé ici

4. Autres bonnes pratiques de sécurité

  • Ne jamais faire confiance aux données utilisateur.

  • Valider et filtrer toutes les entrées.

  • Mettre à jour régulièrement PHP et bibliothèques.

  • Utiliser HTTPS.

  • Configurer correctement les permissions du serveur.
