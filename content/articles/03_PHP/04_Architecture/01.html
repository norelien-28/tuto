3.2.10 – Organisation du code : fichiers, includes et MVC simple

1. Pourquoi organiser son code ?

Un code bien organisé est plus facile à maintenir, comprendre et faire évoluer.
Cela évite les répétitions et améliore la collaboration.

2. Utiliser les fichiers et includes

Inclure un fichier PHP

php
include 'header.php';    // Warning en cas d'erreur, continue l'exécution
require 'config.php';    // Fatal error en cas d'erreur, stop l'exécution

Inclure une fois (éviter les inclusions multiples)

php
include_once 'fonctions.php';
require_once 'config.php';

3. Structure de fichiers classique

bash
/mon_projet
│
├── index.php          # Point d'entrée de l'application
├── config.php         # Configuration (base de données, constantes)
├── header.php         # En-tête HTML commun
├── footer.php         # Pied de page HTML commun
├── fonctions.php      # Fonctions PHP réutilisables
├── css/               # Feuilles de style
├── js/                # Scripts JavaScript
└── pages/             # Pages spécifiques

4. Introduction au modèle MVC simple

MVC = Modèle – Vue – Contrôleur

Composant Rôle
Modèle Gestion des données (ex : base de données)
Vue Affichage et interface utilisateur
Contrôleur Logique métier, réception des actions utilisateur

5. Exemple simplifié

  • index.php (contrôleur) :

php
<?php
require 'model.php';
require 'view.php';

$data = getDataFromDatabase();
renderView($data);

  • model.php :

php
<?php
function getDataFromDatabase() {
    // Connexion + requête SQL
    return ['item1', 'item2', 'item3'];
}

  • view.php :

php
<?php
function renderView($data) {
    foreach ($data as $item) {
        echo "<p>$item</p>";
    }
}

6. Avantages du MVC

  • Séparation claire des responsabilités

  • Facilite la maintenance et l’évolution

  • Permet le travail collaboratif

  • Base pour utiliser des frameworks modernes
