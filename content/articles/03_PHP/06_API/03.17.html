3.2.17 – Sécurité et authentification des API (JWT, OAuth)

1. Pourquoi sécuriser une API ?

Une API non sécurisée peut être :

  • Exploitée par des utilisateurs non autorisés

  • Sujette à des injections (SQL, XSS)

  • Cible d’attaques par force brute, sniffing, etc.

Sécuriser une API signifie :

  • Vérifier l'identité du client

  • S'assurer des permissions (authentification + autorisation)

  • Protéger les données sensibles

2. Authentification par Token

Contrairement aux sessions (stockées côté serveur), les tokens sont portables et stateless : ils sont envoyés avec chaque requête (souvent dans l’en-tête Authorization).

Exemple d’en-tête :

makefile
Authorization: Bearer <token>

3. JWT (JSON Web Token)

a. Structure d’un JWT

Un token JWT est une chaîne encodée en 3 parties :

css
[Header].[Payload].[Signature]

Exemple (simplifié) :

nginx
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9
.eyJ1c2VySWQiOjEsIm5vbSI6IkFsaWNlIn0
.x3r4uQ_kf04MQLI2WZs_wYMLCz5EuKP2KH4TnGHK7u8

b. Utilisation classique

  1. L'utilisateur s'identifie (POST /login)

  2. Le serveur renvoie un JWT signé

  3. À chaque requête, le client envoie ce JWT

  4. Le serveur le vérifie pour authentifier

4. Exemple simple avec JWT en PHP

⚠️ Il est conseillé d’utiliser une bibliothèque comme firebase/php-jwt 

bash
composer require firebase/php-jwt

Générer un token

php
use Firebase\JWT\JWT;
use Firebase\JWT\Key;

$secretKey = "cle_secrete";
$payload = [
    'iat' => time(),
    'exp' => time() + 3600,
    'user_id' => 1
];

$jwt = JWT::encode($payload, $secretKey, 'HS256');

Vérifier un token

php
try {
    $decoded = JWT::decode($jwt, new Key($secretKey, 'HS256'));
    // Accès accordé
} catch (Exception $e) {
    http_response_code(401);
    echo json_encode(['message' => 'Token invalide']);
    exit;
}

5. OAuth 2.0 (notion plus avancée)

OAuth est un protocole d’autorisation très utilisé (par Google, GitHub, Facebook...).

Cas d’usage

  • Authentification via des services tiers (SSO)

  • Délégation d'accès à des applications clientes

Principaux concepts :

  • Client ID / Secret

  • Token d'accès (access token)

  • Refresh token

  • Scopes (droits limités par domaine)

Implémenter un serveur OAuth complet est complexe et souvent délégué à une lib/framework ou un service externe.

6. Sécuriser les endpoints

a. Forcer l’usage de HTTPS

Toujours chiffrer les communications entre client et serveur (SSL/TLS).

b. Protéger les routes sensibles

php
// Middleware simple
function requireAuth() {
    $headers = getallheaders();
    if (!isset($headers['Authorization'])) {
        http_response_code(401);
        echo json_encode(['message' => 'Token manquant']);
        exit;
    }

    $token = str_replace('Bearer ', '', $headers['Authorization']);
    
    try {
        $decoded = JWT::decode($token, new Key('cle_secrete', 'HS256'));
        // On peut ajouter $decoded dans une variable globale ou une session API
    } catch (Exception $e) {
        http_response_code(401);
        echo json_encode(['message' => 'Token invalide']);
        exit;
    }
}

7. Autres mesures de sécurité importantes

  • ✅ Valider toutes les entrées utilisateurs

  • ✅ Protéger contre les injections SQL (avec PDO + requêtes préparées)

  • ✅ Limiter les requêtes (Rate Limiting)

  • ✅ Implémenter des logs (suspicious activity, erreurs)

  • ✅ Ajouter des rôles et niveaux de permission
