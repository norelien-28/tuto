3.2.7 – Sessions et cookies en PHP

1. Les cookies

Un cookie est un petit fichier stocké sur l’ordinateur de l’utilisateur par le navigateur.

Création d’un cookie

php
setcookie("utilisateur", "Alice", time() + 3600); // valable 1 heure

Lire un cookie

php
if (isset($_COOKIE["utilisateur"])) {
    echo "Bonjour " . $_COOKIE["utilisateur"];
}

Supprimer un cookie

php
setcookie("utilisateur", "", time() - 3600);

2. Les sessions

Une session permet de stocker des données côté serveur, liées à un utilisateur.

Démarrer une session

php
session_start();

Stocker des données dans la session

php
$_SESSION['nom'] = "Alice";

Accéder aux données de session

php
echo $_SESSION['nom'];

Détruire une session

php
session_start();
session_destroy();

3. Différences entre cookies et sessions

Critère Cookie Session
Stockage Navigateur (client) Serveur
Sécurité Moins sécurisé Plus sécurisé
Durée de vie Paramétrable (date d’expiration) Jusqu’à fermeture du navigateur (ou destruction)
Capacité Limité (~4 Ko) Plus grande

4. Bonnes pratiques

  • Ne jamais stocker d’informations sensibles dans les cookies.

  • Utiliser les sessions pour les données sécurisées.

  • Toujours appeler session_start() avant tout affichage (avant tout echo ou balises HTML).

  • Valider et filtrer les données des cookies.

5. Exemple simple de gestion de connexion avec sessions

php
// login.php
session_start();

if ($_POST['username'] === 'admin' && $_POST['password'] === 'secret') {
    $_SESSION['user'] = 'admin';
    header('Location: accueil.php');
    exit();
} else {
    echo "Identifiants incorrects.";
}

php
// accueil.php
session_start();

if (!isset($_SESSION['user'])) {
    header('Location: login.php');
    exit();
}

echo "Bienvenue " . $_SESSION['user'];
