3.2.9 – Requêtes SQL en PHP (CRUD)

1. CRUD : Définition

CRUD signifie :

  • Create : créer des données (INSERT)

  • Read : lire des données (SELECT)

  • Update : modifier des données (UPDATE)

  • Delete : supprimer des données (DELETE)

2. Connexion à la base (exemple avec PDO)

php
$pdo = new PDO(...); // Connexion comme vu dans 3.2.8

3. Create : INSERT

php
$stmt = $pdo->prepare("INSERT INTO utilisateurs (nom, email) VALUES (:nom, :email)");
$stmt->execute([
    ':nom' => 'Jean',
    ':email' => 'jean@mail.com'
]);

4. Read : SELECT

Lire tous les enregistrements

php
$stmt = $pdo->query("SELECT * FROM utilisateurs");
$utilisateurs = $stmt->fetchAll();

foreach ($utilisateurs as $user) {
    echo $user['nom'] . "<br>";
}

Lire un enregistrement par ID

php
$id = 1;
$stmt = $pdo->prepare("SELECT * FROM utilisateurs WHERE id = :id");
$stmt->execute([':id' => $id]);
$user = $stmt->fetch();

if ($user) {
    echo $user['nom'];
}

5. Update : UPDATE

php
$stmt = $pdo->prepare("UPDATE utilisateurs SET email = :email WHERE id = :id");
$stmt->execute([
    ':email' => 'nouvel.email@mail.com',
    ':id' => 1
]);

6. Delete : DELETE

php
$stmt = $pdo->prepare("DELETE FROM utilisateurs WHERE id = :id");
$stmt->execute([':id' => 1]);

7. Sécurité : prévention des injections SQL

  • Toujours utiliser des requêtes préparées (prepare + execute) avec des paramètres liés.

  • Ne jamais concaténer directement des données utilisateur dans les requêtes SQL.

8. Gestion des erreurs

Utiliser try/catch pour attraper les erreurs :

php
try {
    // requête SQL
} catch (PDOException $e) {
    echo "Erreur : " . $e->getMessage();
}
